БД предназначена для хранения данных сети магазинов одежды.
БД разделена на сущности: 1)товары, 2)производители, 3)группы товаров, 4)типы товаров, 5)размеры, 6)цвета,
7)магазины, 8)закупки, 9)продажи, 10)курс обмена валюты, 11)таблица объединающая товар, цвет и рамер
12)таблица объединающая товар, цвет и рамер(id), стоимость такого товара .
Посмотреть структуру таблицы можно на диаграмме.
В БД сделано два представления: Закупки-Магазины и Типы товаров-Товары.
Сделаны примеры индексов для столбцов sales.id_sales,sales.amount использующихся в запросах.
Реализованы скрипты позволяющие: 
Показать полную сумму покупок в магазине;
Показать полную сумму покупок во всех магазинах;
Показать сколько позиций товара было продано и сколько стоит одна позиция такого товара по номеру магазина  + столбец с полной суммой покупки;
Показать (упорядочено по номеру магазина) какой магазин(по номеру) в какой день по какому курсу делал закупку;
Показать (упорядочено по дате) какой магазин(по адресу) в какой день по какому курсу делал закупку;
Показать все типы товаров определенного размера определенной фирмы; 
Показать Id продукта, описание и цвет, сортировка по ID продукта;
Транзакции по добавлению/удалению позиции товара;
Функция считающая стоимость закупки в долларах по ее id;
Триггер запрещающий добавлять NULL значения name или address в таблицу stores.
